# Concept C - Data-first Coaching Engine

## Description
Prioritize backend redesign first: rich domain model, rules engine, and event/audit trails, while keeping UI mostly incremental until APIs stabilize.

## Pros
- Strong long-term technical foundation.
- Lower risk of core logic rewrites.
- Better testability and security posture early.

## Cons
- Slower visible user experience improvements.
- Product may still feel like PoC during early sprints.
- Harder to validate UX assumptions quickly.

## Estimated Effort
- Sprint 1-2: backend contexts and migration.
- Sprint 3+: full UI transformation.
